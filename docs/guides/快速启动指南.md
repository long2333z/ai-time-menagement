## Windows用户

### 1️⃣ 一键启动

双击运行 `start.bat` 文件，脚本会自动：
- ✅ 检查Node.js和Python环境
- ✅ 安装前端和后端依赖
- ✅ 启动前端服务（端口3000）
- ✅ 启动后端服务（端口8000）

### 2️⃣ 访问应用

启动成功后，浏览器访问：
- **前端页面**: http://localhost:3000
- **后端API**: http://localhost:8000
- **API文档**: http://localhost:8000/docs

### 3️⃣ 停止服务

双击运行 `stop.bat` 文件，会自动停止所有服务。

首次使用需要添加执行权限：

## 🔧 常见问题

### Q1: 启动失败，提示找不到 Node.js 或 Python？

**解决方案**：
1. 确保已安装 Node.js 18+ 和 Python 3.10+
2. 重启终端或命令提示符
3. 检查环境变量是否正确配置

### Q2: 端口被占用怎么办？

**Windows**：
```bash
# 查看占用端口的进程
netstat -ano | findstr :3000
netstat -ano | findstr :8000

# 结束进程（替换 PID 为实际进程ID）
taskkill /F /PID <PID>
```

**Linux/Mac**：
```bash
# 查看占用端口的进程
lsof -i :3000
lsof -i :8000

# 结束进程（替换 PID 为实际进程ID）
kill -9 <PID>
```

### Q3: 依赖安装失败？

**前端依赖**：
```bash
# 清除缓存重新安装
rm -rf node_modules package-lock.json
npm install
```

**后端依赖**：
```bash
# 使用国内镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q4: 如何查看日志？

**前端日志**：在运行 `npm run dev` 的终端窗口中查看

**后端日志**：
- 终端输出：在运行后端服务的终端窗口中查看
- 日志文件：`backend/logs/ai_time_management.log`
- 错误日志：`backend/logs/ai_time_management_error.log`

### Q5: 数据存储在哪里？

数据库文件位置：`backend/data/ai_time_management.db`

这是一个 SQLite 数据库文件，可以直接备份。

### Q6: 如何重置数据？

**方法1**：删除数据库文件
```bash
# Windows
del backend\data\ai_time_management.db

# Linux/Mac
rm backend/data/ai_time_management.db
```

**方法2**：使用管理员账户登录，在设置中重置

重启服务后会自动创建新的数据库。

### Q7: 默认管理员账户是什么？

- **用户名**: `admin`
- **密码**: `admin123`

⚠️ **首次登录后请立即修改密码！**
