## Windows用户

### 1️⃣ 一键启动

双击运行 `start.bat` 文件，脚本会自动：
- ✅ 检查Node.js和Python环境
- ✅ 安装前端和后端依赖
- ✅ 启动前端服务（端口3000）
- ✅ 启动后端服务（端口8000）

### 2️⃣ 访问应用

启动成功后，浏览器访问：
- **前端页面**: http://localhost:3000
- **后端API**: http://localhost:8000
- **API文档**: http://localhost:8000/docs

### 3️⃣ 停止服务

双击运行 `stop.bat` 文件，会自动停止所有服务。

## Linux/Mac用户

### 1️⃣ 添加执行权限

首次使用需要添加执行权限：
```bash
chmod +x start.sh stop.sh
```

运行启动脚本：

## 📦 系统要求

### 必需软件
- **Node.js**: 18.0 或更高版本
  - 下载地址: https://nodejs.org/
- **Python**: 3.10 或更高版本
  - 下载地址: https://www.python.org/

### 端口要求
- 前端端口: `3000`
- 后端端口: `8000`

确保这两个端口未被占用。

## 🔧 常见问题

### Q1: 提示"命令未找到"或"不是内部或外部命令"

**原因**: Node.js或Python未安装，或未添加到系统PATH

**解决方法**:
1. 确认已安装Node.js和Python
2. 重新安装时勾选"Add to PATH"选项
3. 重启命令行窗口

### Q2: 端口被占用

**错误信息**: `端口 3000/8000 已被占用`

**解决方法**:
1. 停止占用端口的程序
2. 或修改配置使用其他端口

**Windows查看端口占用**:
```cmd
netstat -ano | findstr :3000
netstat -ano | findstr :8000
```

**Linux/Mac查看端口占用**:
```bash
lsof -i :3000
lsof -i :8000
```

### Q3: 依赖安装失败

**解决方法**:

**前端依赖**:
```bash
# 清理缓存
npm cache clean --force
# 删除node_modules
rm -rf node_modules
# 重新安装
npm install
```

**后端依赖**:
```bash
cd backend
# 删除虚拟环境
rm -rf venv
# 重新创建
python -m venv venv
# 激活并安装
source venv/bin/activate  # Linux/Mac
# 或
venv\Scripts\activate.bat  # Windows
pip install -r requirements.txt
```

### Q4: 启动后页面无法访问

**检查步骤**:
1. 确认服务是否启动成功
2. 检查防火墙设置
3. 查看日志文件：
   - 后端日志: `logs/backend.log`
   - 前端日志: `logs/frontend.log`

### Q5: 数据库连接失败

**默认使用SQLite**，无需额外配置。

如需使用MySQL，请参考 [QUICK_START.md](./QUICK_START.md) 中的MySQL配置说明。

## 📝 手动启动（高级用户）

如果自动脚本无法使用，可以手动启动：

### 启动后端
```bash
cd backend
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或
venv\Scripts\activate.bat  # Windows
pip install -r requirements.txt
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 启动前端
```bash
npm install
npm run dev
```

## 💡 提示

- 首次启动会自动安装依赖，需要一些时间
- 启动后会打开两个命令行窗口（前端和后端）
- 关闭窗口即可停止对应服务
- 建议使用 `stop.bat` 或 `stop.sh` 优雅停止服务
